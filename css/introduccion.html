CSS: Introducción

Historia

CSS (Cascading Style Sheets) es un lenguaje que permite describir la apariencia y el formato de un documento escrito en un lenguaje de markup, como el HTML.

CSS hizo su aparición en 1994. En esa época, los fabricantes de navegadores comenzaron a incorporar elementos de estilo en el HTML para responder a las demandas de los desarrolladores, aumentando su complejidad y dificultando la compatibilidad y estandarización. CSS nace con el objetivo de corregir este proceso, reduciendo la complejidad y mejorando la interoperabilidad.

La primera recomendación de la W3C fue publicada en 1996. CSS 2, la segunda recomendación hizo su aparición al año siguiente, en 1997.

La adopción de CSS ha sido un proceso lento, con algunos navegadores implementando sus propias reglas y extensiones. En los primeros años de CSS era común el uso incorrecto de HTML para dar estilo a las páginas y la creación de sitios optimizados para un navegador específico, debido a la falta de estandarización

CSS 3

A pesar de esto, el desarrollo de CSS continuó y lentamente el soporte de la mayoría de los navegadores convergió hacia las recomendaciones de la W3C. Los fabricantes han continuado incorporando nueva funcionalidad que ha ido convirtiéndose en CSS 3, la tercera recomendación de la W3C.

CSS 3 no está completa, muchas de sus secciones continúan como borrador. Sin embargo, hay un conjunto importante de características soportadas por todos los navegadores modernos que pueden usarse de forma segura en los sitios actuales.

Selectores

Los selectores permiten especificar a cual elemento de HTML se aplicará una serie de reglas de estilo específicas. Un selector puede contener nombres de etiquetas HTML, ID, clases u otros elementos, en diversas combinaciones.

Los selectores son evaluados en cascada, o sea, de forma secuencial. Si un selector se repite, el último sobreescribirá las reglas aplicadas por el anterior.

Por ejemplo, considere el siguiente fragmento de HTML:

<ul class="lista-compras">
  <li>Arroz</li>
  <li id="importante">Tocineta</li>
  <li>Jabón para ropa <strong>Rinso</strong></li>
  <li>Leche (<span class="aclaracion">Descremada</span>)</li>
  <li>Lavaplatos</li>
</ul>

Si se aplican los siguientes estilos:

li {
  color: blue;
}
…
li {
  color: red;
}

El color final de los ítems de lista será rojo.

Pueden usarse nombres de clases y ID como selectores. Para usar una clase, el nombre debe ser precedido por un punto. En el caso de los ID, debe usarse el signo de número (#).

.lista-compras {
  font-weight: normal;
}

#importante {
  font-weight: bold;
}

En la mayoría de los casos, las reglas aplicadas sobre un elemento son heredadas por los elementos hijos, siempre y cuando no haya una regla más específica aplicada sobre estos.

Es posible anidar selectores, para indicar una jerarquía. Por ejemplo la siguiente regla:

.lista-compras li strong {
  background-color: yellow;
}

Aplica la regla background-color al elemento strong que está dentro de un li y que a su vez está dentro de un elemento con la clase lista-compras.

Especificidad

Es posible utilizar distintos selectores para referirse a un mismo elemento. En el ejemplo anterior, pueden usarse los siguientes selectores para referirse al segundo ítem de la lista.

  • li
  • ul li
  • .lista-compras li
  • #importante

Para determinar cúal de los selectores tiene prioridad sobre los otros, existe una medida conocida como especificidad. Si dos selectores pueden aplicarse al mismo elemento, la prioridad la tendrá el selector que tenga la mayor especifidad.

Para evaluar la especificidad de un selector, debe evaluarse la cantidad de elementos que tenga en cada una de las siguientes categorías, ordenadas de la menor especificidad a la mayor:

  • Etiquetas HTML y pseudo elementos (ej. :before) [1x]
  • Clases, selectores de atributo(ej. [type="checkbox"]) y pseudo clases (ej. :hover) [10x]
  • ID [100x]
  • Estilos inline [1000x]
    • .

En el caso de los selectores del ejemplo, las especificidades serán las siguientes:

  • li: 1 x 1 = 1
  • ul li: 2 x 1 = 2
  • .lista-compras li: 1 x 1 + 1 x 10 = 11
  • #importante: 1 x 100 = 100

Por lo tanto, los estilos aplicados usando el selector #importante tendrán la mayor prioridad.

Ante la duda, puede usarse una calculadora de especificidad.
